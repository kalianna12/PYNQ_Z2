paths used by this project. Use it when moving the project to another PC.

## 4. Python / Notebook Register Rule

When writing or updating PYNQ Python scripts and `.ipynb` notebooks, do not
guess AXI-Lite register offsets from experience.

Priority order:

```text
1. Use real addresses from generated HLS/Vivado files and logs.
2. Prefer register_map names when the .hwh exposes them correctly.
3. Only use hardcoded ip.write/ip.read offsets after confirming them from
   generated files.
```

Useful generated files:

```text
hls/base_add_prj/solution1/impl/misc/drivers/base_add_v1_0/src/xbase_add_hw.h
pynq/base_add.hwh
HLS_REPORT.md
VIVADO_OVERLAY_REPORT.md
vivado.log
vivado_hls.log
```

Example: if `xbase_add_hw.h` says:

```c
#define XBASE_ADD_CTRL_ADDR_BUFFER_R_DATA     0x10
#define XBASE_ADD_CTRL_ADDR_SAMPLE_COUNT_DATA 0x18
```

then Python may use:

```python
ip.write(0x10, buffer.physical_address)
ip.write(0x18, sample_count)
```

If the HLS function ports or pragmas change, rerun HLS first and re-check these
generated addresses before editing Python or notebooks.

These board output files are useful to keep if you want the example to run on
PYNQ immediately:

```text
pynq/base_add.bit
pynq/base_add.hwh
```

## 8. HLS Tcl Flow

Script:

```text
hls/hls.tcl
```

Main steps:

```text
open_project -reset base_add_prj
set_top base_add
add_files src/base_add.cpp
add_files src/base_add.h
add_files -tb tb/test_base_add.cpp
open_solution -reset solution1
set_part xc7z020clg400-1
create_clock -period 10
csim_design
csynth_design
export_design -format ip_catalog
```

The script also applies a Vivado 2018.2 workaround for modern dates when IP
export hits a `core_revision` lexical cast error. This is normal for old Vivado.

## 9. Vivado Tcl Flow

Script:

```text
vivado/build.tcl
```

Main steps:

```text
create_project base_add_overlay
set board_part tul.com.tw:pynq-z2:part0:1.0
add HLS IP repo from hls/base_add_prj/solution1/impl/ip
create Zynq processing_system7
create base_add_0 HLS IP
connect PS M_AXI_GP0 to base_add_0/s_axi_CTRL
assign_bd_address
generate block design
run synthesis
run implementation
write bitstream
copy bit/hwh to pynq/
```

## 10. PYNQ Board Files

Upload these to the PYNQ board after rebuilding the overlay:

```text
pynq/base_add.bit
pynq/base_add.hwh
pynq/base_add_test.py
pynq/base_add_demo.ipynb
```

On the board, run:

```bash
python3 base_add_test.py
```

Or open in browser Jupyter:

```text
http://192.168.2.99:9090
```

Then open:

```text
base_add_demo.ipynb
```

VS Code's Jupyter extension is optional. The PYNQ board's browser Jupyter is
enough for running the notebook.

## 12. Quick Porting Checklist

1. Install Vivado/Vivado HLS 2018.2.
2. Clone or copy this project folder.
3. Update Vivado paths if the install location changed.
4. Open project root in VS Code.
5. Run `FPGA: 1 Build HLS IP`.
6. Check `HLS_REPORT.md`.
7. Run `FPGA: 2 Build Vivado Overlay`.
8. Check `VIVADO_OVERLAY_REPORT.md`.
9. Upload `pynq/base_add.bit`, `pynq/base_add.hwh`, and Python/Jupyter files to PYNQ.
10. Run `python3 base_add_test.py` on the board.
